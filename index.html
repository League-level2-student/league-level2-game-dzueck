Running Failure

by: Dylan Zueck

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I wanted to make a game that did not use the keyboard or mouse as the movement often feels bad. I thought the mouse would be fun to play a game using.

2. What I Learned

They mentioned it did not scale in difficulty over time and that it was too sudden of a start.

3. Architecture of My Game

The runner uses acceses the mouse x and y position to track it. The nerd square does the same for the runner. The bouncing nerd square changes it's angle as it hits walls randomizing it a bit so it is not super predictable.

4. Technical Implementation

To get the runner to track the mouse, I had to get the x and y of the mouse and find the angle a line between the runner and the mouse would make. Once I have the angle I use geometry to change the x and y position. I also had to make it work if you move the game window as the x and y position of the mouse is relative to the screen not the window. I solved this by using the x and y position of the JFrame.

5. Most Challenging Part

The most challenging part was creating the base framework such as the ObjectManager. I used the league invaders to help me with this part.

6. Future Fixes and Updates

I would lower the speed increase of the nerd square based on your score as to allow you to get a higher score overall.
